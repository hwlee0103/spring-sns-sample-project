## Task

code-reviewer agent 를 실행하여 코드리뷰 해.
현재 변경된 파일이 없을 경우 전체 소스 대상으로 코드리뷰 진행해.
지금까지 만든 코드에서 캐시, 쿠키, 토큰, 에러 처리 네 가지 관점에서 민감 정보 노출 가능한 곳 찾고 코드리뷰 내용으로 추가해.
(수정, 개선이 필요한 부분, 권장사항 포함)

리뷰 진행 후 `.claude/review-history/` 디렉토리 내 **월 단위 파일**(`YYYY-MM.md`)을 다음 규칙에 따라 갱신해줘.

### 파일 명명 규칙
- 히스토리 파일은 일자별이 아닌 **월 단위**로 관리한다. (예: `2026-04.md`, `2026-05.md`)
- 신규 파일이 필요하면 동일한 표 형식과 상태값 정의 섹션을 포함하여 생성한다.

### 갱신 규칙

리뷰 결과를 적용할 때 반드시 아래 3가지 케이스를 처리한다.

1. **기존 이슈가 해결된 경우**
   - 이슈가 **최초 발견된 월 파일**(예: `2026-04.md`)에서 해당 행의 `상태`를 `검증 및 소스 재 리뷰 완료`로 변경하고 `해결 일자`(오늘 날짜)를 기록한다.

2. **해결된 이슈를 해결 월 파일에도 기록**
   - 이슈가 **해결된 월의 파일**(예: 4월 발견 → 5월 해결이면 `2026-05.md`)에 해결 내역을 별도 섹션(`## Resolved (from YYYY-MM)`)으로 추가한다.
   - 동일 월에서 발견·해결된 경우(예: 4월 발견 → 4월 해결)는 `2026-04.md` 내에서 상태/해결 일자만 갱신하면 충분하므로 별도 Resolved 섹션을 만들지 않는다.

3. **신규 이슈 발견 시**
   - 오늘 날짜가 속한 월의 파일(예: `2026-05-03` → `2026-05.md`)에 신규 Review 섹션으로 추가한다.
   - 해당 월 파일이 없으면 새로 생성한다.

### 표/상태 형식 (모든 월 파일 공통)

| # | 구분 | 파일 | 위치 | 이슈 내용 | 상태 | 해결 일자 |
|---|------|------|------|-----------|------|-----------|

상태값:

| 상태 | 설명 |
|------|------|
| detect | 이슈 최초 발견 |
| 수정 진행중 | 수정 작업 진행 중 |
| 수정 완료 | 수정 작업 완료 |
| 검증 및 소스 재 리뷰 완료 | 수정 검증 및 재 리뷰 완료 |
